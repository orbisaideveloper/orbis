# Purpose

This document defines the official vision, mission, philosophy, objectives and long-term direction of the Orbis project.

It serves as the project's constitution and provides the foundation for all architectural, development and business decisions.

Every AI model, developer and contributor should understand this document before working on the project.

---

# Project Vision

Build Orbis into a unified AI Software Factory and AI Orchestration Platform that enables anyone to transform business ideas into production-ready software.

---

# Project Mission

Simplify software development by allowing users to describe their business requirements while Orbis intelligently coordinates planning, architecture, AI models, developer tools, testing, quality verification and deployment.

# What is Orbis?

Orbis is an AI Software Factory and AI Orchestration Platform.

It intelligently coordinates AI models, developer tools and cloud services to transform business ideas into production-ready software.

Instead of requiring users to manually manage multiple platforms, Orbis orchestrates the complete software development lifecycle through one unified workspace.

---

# Project Philosophy

Technology should never become a barrier to innovation.

Users should focus on their business ideas, not on development tools.

Orbis coordinates AI models, developer tools and cloud services so users can focus on solving real business problems.

Instead of learning multiple platforms, configuring complex environments and managing different services, users simply describe what they want to build.

Orbis manages the technology.

Users build the business.

---

# What Orbis is NOT

Orbis is NOT an AI model.

Orbis is NOT a replacement for ChatGPT, Claude or Gemini.

Orbis is NOT a traditional IDE.

Its purpose is orchestration, automation and software project management.

# Design Principles

- GitHub is the Single Source of Truth.
- Every AI must follow the documentation.
- Human approval is required for major architectural changes.
- The platform must remain modular.
- AI providers should be replaceable.
- Every important action should be traceable.
