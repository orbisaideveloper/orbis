# Document Scope

This document defines how Orbis manages subscriptions, platform usage, billing and revenue.

This document does NOT define:

• Database Structure

• Payment Gateway Implementation

• Security Policies

• API Provider Configuration

Those topics are documented separately.

---

# Purpose

The purpose of this document is to create a transparent, scalable and sustainable business model for the Orbis platform.

Users should always understand what they are using and why they are being charged.

---

# Business Philosophy

Value First.

Payment Second.

Users should experience the platform before purchasing a subscription.

---

# Free Experience

New users receive a limited free experience based on platform usage.

The free experience is controlled by usage limits rather than only by time.

Projects remain safe even after free usage ends.

---

# Subscription Plans

• Free Experience

• Starter

• Professional

• Business

• Enterprise

Plan details may evolve over time without changing the platform architecture.

---

# Usage Based Billing

Platform billing is based on measured usage.

Examples include:

• AI Requests

• AI Token Usage

• Build Count

• Preview Count

• Storage Usage

• Project Count

• Team Members

• Deployment Usage

---

# Usage Ledger

Every platform activity is recorded inside the Usage Ledger.

Examples:

• AI Chat

• AI Build

• Image Generation

• Voice Processing

• Document Analysis

• Preview

• Deployment

• Storage

The Usage Ledger becomes the official source for billing calculations.

---

# Billing Engine

Billing is calculated using:

↓

Usage Ledger

↓

Subscription Rules

↓

Billing Engine

↓

Invoice

↓

Payment

No billing should rely on estimation alone.

---

# User Identity

Every registered user receives a permanent Orbis User ID.

The User ID remains unchanged regardless of role or subscription.

All billing, usage, projects and subscriptions are linked to the same User ID.

---

# Cost Management

Platform cost should include:

• AI Provider Cost

• Infrastructure Cost

• Storage Cost

• Platform Operating Cost

• Support Cost

• Business Margin

Customer pricing should always remain sustainable.

---

# User Dashboard

Users should be able to view:

• Current Plan

• Remaining Usage

• Current Month Usage

• Billing History

• Invoice History

• Upgrade Options

---

# Admin Dashboard

Administrators should monitor:

• Total Revenue

• Total AI Cost

• Provider-wise Cost

• User-wise Usage

• Project-wise Usage

• Monthly Profit

• Platform Health

---

# Upgrade Experience

Upgrade requests should appear naturally.

Users should never lose access to their existing projects.

When free usage ends:

• Existing Projects remain accessible.

• New premium operations require a subscription.

---

# Billing Principles

Measured Usage

Transparency

Fair Pricing

Provider Independence

Complete Auditability
