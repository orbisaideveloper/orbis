# AC-004 — Universal Reporting Standard

**Project:** ORBIS

**Layer:** Modules → Accounting

**Document ID:** AC-004

**Version:** 1.0

**Status:** Draft for Approval

---

# Purpose

This document defines the standard reporting framework used by every accounting-enabled module within the ORBIS Platform.

The reporting engine is shared across DigiLedger, Farmer Brain and all future accounting modules.

---

# Vision

One Reporting Engine.

Unlimited Business Modules.

Every module produces reports using the same reporting standards.

Only business-specific report layouts may differ.

---

# Universal Report Types

Every accounting module should support:

- Statement
- Ledger
- Cash Book
- Outstanding Report
- Customer Summary
- Supplier Summary
- Daily Report
- Weekly Report
- Monthly Report
- Yearly Report
- Profit & Loss (where applicable)
- Inventory Report

---

# Standard Export Formats

Every report should support:

- PDF
- Excel (XLSX)
- CSV
- Print

Future:

- JSON Export
- API Export

---

# Universal Report Structure

Every report should contain:

- Report Name
- Business Name
- Module Name
- ORB-ID
- Generated Date & Time
- Date Range
- Generated By
- Version

---

# Real-Time Reporting

Reports should always reflect the latest available accounting data.

Manual recalculation is not required.

---

# Filters

Reports should support filtering by:

- Date
- Customer
- Supplier
- Transaction Type
- Payment Status
- Module

---

# Report Integrity

Reports are generated from Ledger records.

Reports never modify accounting data.

Reports are read-only outputs.

---

# Report Sharing

Reports may be:

- Printed
- Downloaded
- Shared
- Archived

Only authorized users may access reports.

---

# Design Principles

One Reporting Standard

Real-Time

Consistent

Printable

Exportable

Future Scalable
